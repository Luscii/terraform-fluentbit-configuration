# .NET Logging Example

## Purpose

Demonstrates how to configure Fluent Bit for .NET application logging using this module. This example sets up the required label context and log source for a .NET container, and outputs the generated parser and filter configuration.

## Prerequisites

- AWS Account with appropriate permissions
- Terraform >= 1.3
- Existing ECS/Fargate cluster with a .NET container (e.g., `dotnet-app`)

## Usage

1. Update variables in `terraform.tfvars` or provide via CLI:
   ```bash
   terraform plan -var="dotnet_container=dotnet-app"
   ```
2. Review the plan:
   ```bash
   terraform plan
   ```
3. Apply the configuration:
   ```bash
   terraform apply
   ```

## Inputs

See [variables.tf](variables.tf) for all available inputs.

- `namespace` - Namespace for resource naming
- `environment` - Environment name
- `name` - Name for the label and resources
- `dotnet_container` - Name of the .NET container in ECS

## Outputs

See [outputs.tf](outputs.tf) for all outputs.

- `dotnet_parsers` - The .NET parser configuration generated by the module
- `dotnet_filters` - The .NET filter configuration generated by the module

## Cleanup

To destroy all resources created by this example:

```bash
terraform destroy
```

## Notes

- This example does not deploy an ECS service; it only demonstrates Fluent Bit configuration for .NET logs.
- For more details, see [ADR-0006](../../docs/adr/0006-dotnet-pending-implementation.md) and the main [README.md](../../README.md).
